## 3. 과제 목표
- 조건 분기의 용어나 방법이 툴마다 상이할 수 있다는 부분을 언급하면 좋을 듯 보임 (Make의 경우 Flow Control, Zapier의 경우 Flow Controls 내에 다양하게 존재함)

## 5. 보너스 과제
- 생성형 AI를 Action으로 추가해 사용하려면 API Key나 자동화 플랫폼 상의 토큰 등이 필요할 수 있고 과금 가능성이 있다는 부분을 언급해도 좋을 것 같음
- API Key에 대해 안내하기 전에 [API가 무엇인지](https://namu.wiki/w/API) 학습하도록 유도하고, 하단 `6. 제약 사항`에 나오는 API Key는 외부에 유출되면 안되는 중요한 정보라는 것을 강조하여 안내하는 것이 좋을 것 같음

## 6. 제약 사항
- `프로젝트 2는 Trigger 발생 시 자동 실행되어야 한다.` -> Make 툴의 무료 플랜의 경우 실시간으로 Trigger를 감지하지 않고 15분에 한번씩 감지한다는 언급 필요
- [Webhook에 대해 간단한 개념](https://docs.tosspayments.com/resources/glossary/webhook)을 확인하고 진행하도록 유도하는 것이 좋을 것 같음
